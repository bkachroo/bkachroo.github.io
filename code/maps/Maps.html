mapGen makes the lookup table for a particular map from the index. The padding makes sure it works for smaller indices. Look up by selecting the index of the list corresponding to the integer form of the input sequence. Imagine: put the right number into this function and it will give you the solution to every traveling salesman problem of a certain size. Maps are at one extreme of the continuum between memory and processing. No processing required except lookup, but also barely any compression. This suggests a trade-off between compression and processing, as though processing is just decompression. Below: first 10 2-2 maps
